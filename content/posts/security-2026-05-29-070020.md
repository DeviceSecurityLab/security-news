### [New Gogs zero-day flaw lets hackers get remote code execution](https://www.bleepingcomputer.com/news/security/new-gogs-zero-day-flaw-lets-hackers-get-remote-code-execution/)
**2026-05-28 23:25 JST ／ BleepingComputer ／ MEDIUM**

**要約:** セルフホスト型 Git サービス Gogs（バージョン 0.14.2 および 0.15.0+dev）に未修正のクリティカルなゼロデイが発見された。Merge() コードパスにおける引数インジェクションの脆弱性で、認証済み攻撃者が悪意あるブランチ名経由で git rebase 操作に `--exec` フラグを注入できる。認証が必要だが、Gogs はデフォルトでオープン登録が有効な場合が多い。2,400台超の Gogs サーバーがインターネットに露出している。

**なぜ重要か:** 成功すればリモートコード実行（RCE）、認証情報窃取（SSH 鍵・API トークン）、全リポジトリへのアクセスが可能となる。オープン登録の常態化により「認証必須」の前提が実質的な防御にならない。

**影響:** 主にアジア・欧州で稼働する2,400台超の露出サーバー。ソースコードと CI/CD 認証情報の窃取を通じたサプライチェーン侵害のリスクがある。

**次のアクション:** インターネット露出している Gogs インスタンスを直ちに洗い出し、オープン登録を無効化、アクセスを VPN/IP 制限の背後に隔離する。ベンダーからのパッチ公開を監視し、SSH 鍵・API トークンのローテーションを検討する。
