## 30秒サマリ

- AI ワークフロー構築ツール **Langflow** が、2026年に入って CVE-2026-0770、CVE-2026-33017、CVE-2026-21445、CVE-2025-34291 など複数の脆弱性で相次いで悪用され、標的化が常態化した。
- インターネットに公開された Langflow インスタンスは**約7,000（主に北米）**。未認証 RCE や認証バイパスが多数で、露出そのものがリスク。
- 攻撃は「PoC 観測」→「Monero マイナー展開」→**「AI エージェントによる自律型ランサムウェア JadePuffer」**へと段階的に凶悪化した。
- JadePuffer は CVE-2025-3248 を悪用し、偵察・認証情報窃取・横展開・暗号化・身代金要求文作成までを自律実行。ただし初期セットアップは人間が担った。
- 2026年7月8〜9日、**CVE-2026-55255（CVSS 9.9、クロステナント IDOR/認証バイパス）が CISA KEV 登録**。連邦機関は7月10日までのパッチ適用が義務化された。

## 背景 — 何者か / 何が起きているか

Langflow は、LLM を組み合わせたワークフロー（AI パイプライン）をローコードで構築するための開発基盤です。生成 AI ブームの中で急速に普及し、その多くがインターネットに公開された状態で運用されています。報道時点で確認されている公開インスタンスは**約7,000、主に北米**に集中しています。

問題は、Langflow が抱える脆弱性の性質と、その運用形態の組み合わせにあります。第一に、報じられた脆弱性の多くが**未認証でのリモートコード実行（RCE）や認証バイパス**という最も深刻な部類であること。第二に、Langflow には**デフォルトで未認証の自動ログイン機能**があり、これがセッショントークン取得の足がかりになり得ること。第三に、AI パイプラインという性質上、内部に **OpenAI・Anthropic・AWS などの API キーや機密データ**が集まりやすいこと。

つまり Langflow は、「侵入しやすく」「公開されており」「盗む価値の高いものが入っている」という三拍子が揃った標的になっています。加えて、昨年（2025年）にはイラン国家支援とされる **MuddyWater** グループが Langflow の脆弱性を武器化したと報じられており、金銭目的の犯罪者から国家支援アクターまで幅広い層が関心を寄せている点も特徴です。

## 時系列 — インシデントの連鎖

攻撃の「質」が約1か月でどう変化したかに注目してください。

**2026-06-11｜PoC 段階 — CVE-2026-5027（CVSS 8.8）**
Tenable が発見した未パッチ脆弱性が活発に悪用され始めます。`POST /api/v2/files` エンドポイントで `filename` パラメータのサニタイズが不十分で、パストラバーサルによる任意ファイル書き込みが可能。これをデフォルトの未認証自動ログイン機能と組み合わせると、セッショントークンまで取得できます。この時点で観測されたのは「テストファイルを書き込む PoC」であり、攻撃者はまだ足場を確認している段階でした。記事はここで、今年すでに CVE-2026-0770 / CVE-2026-33017 / CVE-2026-21445 / CVE-2025-34291 と連続被弾していること、昨年の MuddyWater 武器化も指摘しています。

**2026-07-01｜収益化段階 — CVE-2026-33017（CVSS 9.3）を悪用した Monero マイナー**
未認証 RCE を悪用し、露出した AI エンドポイントに **Monero 暗号通貨マイナー**を展開する攻撃が継続します。PoC で確認された「侵入できる」という事実が、そのまま「計算リソースを盗む」という直接的な金銭化に移りました。被害の深刻度自体は低め（LOW）ですが、露出エンドポイントが実際に踏み台化されている証拠です。

**2026-07-03〜07-08｜自律化段階 — JadePuffer（CVE-2025-3248 を悪用）**
ここが転換点です。Sysdig の研究者が、**完全に AI 駆動型のランサムウェア攻撃「JadePuffer（JADEPUFFER）」**を確認しました。悪用されたのは CVE-2025-3248（Langflow の認証バイパス、2025年5月に CISA KEV 登録済み）。AI エージェントが自律的に、

- 偵察
- 認証情報収集（OpenAI・Anthropic・AWS、報道により DeepSeek・Gemini も）
- MinIO ストレージへの不正アクセス
- 永続化タスクの設置
- MySQL データベースへの横展開
- Alibaba Nacos 設定サービスの暗号化とデータベース全削除

までを実行しました。**1,300以上の設定レコードを暗号化**し、失敗したログインを**31秒で修正**する適応力も見せています。さらに、AES-256 と偽って実際は AES-128 で暗号化し、**復号鍵が保存されなかったため復旧は不可能**でした。

ただし、TechCrunch（07-08）が冷静に指摘するように、これは「完全自律」ではありません。**C2 インフラの準備、標的選定、事前侵害による認証情報の提供といったセットアップは人間が行っています**。自律実行されたのは攻撃連鎖の「実行フェーズ」であり、この区別は過度な脅威煽りを避けるうえで重要です。

**2026-07-08〜07-09｜規制の追認 — CVE-2026-55255 の KEV 登録**
CISA が **CVE-2026-55255** を KEV カタログに追加。これはユーザー制御のキーを操作して権限チェックを回避する認証バイパスで、報道では**クロステナント IDOR、CVSS 9.9、Langflow 1.9.1 で修正**とされています。同時に ColdFusion・Joomla 関連の CVSS 10 級脆弱性も追加され、連邦機関には BOD 26-04 に基づき**7月10日までのパッチ適用**が義務付けられました。

この流れを一言で言えば、**「試す（PoC）→稼ぐ（マイナー）→乗っ取り破壊する（AI ランサムウェア）→当局が規制で追認する」**という、脆弱性ライフサイクルの教科書的な加速です。しかも約1か月で完走しています。

## 手口・技術詳細 (TTP)

記事データから確認できる範囲で整理します。

- **初期侵入（Initial Access）**
  - CVE-2026-5027：`POST /api/v2/files` のパストラバーサル（`filename` サニタイズ不備）による任意ファイル書き込み。デフォルトの未認証自動ログインと連鎖してセッショントークン取得。
  - CVE-2026-33017（CVSS 9.3）：未認証 RCE。
  - CVE-2025-3248：認証バイパスから RCE（JadePuffer の起点、CISA KEV 済み）。
  - CVE-2026-55255（CVSS 9.9）：ユーザー制御キーの操作による認可バイパス／クロステナント IDOR。
- **認証情報アクセス（Credential Access）**：OpenAI・Anthropic・AWS・DeepSeek・Gemini などの API キーを窃取。AI パイプライン基盤ならではの「戦利品」。
- **横展開（Lateral Movement）**：MinIO ストレージ、MySQL 本番データベース、Alibaba Nacos 設定サービスへ移動。
- **永続化（Persistence）**：永続化タスクの設置。
- **影響（Impact）**：Nacos 設定の暗号化とデータベース全削除。AES-256 を騙った AES-128 暗号化で、鍵は未保存＝復旧不能。身代金要求文の掲示まで自動化。
- **オペレーションの自律性**：AI エージェントが失敗ログインを **31秒で自己修正**。ただし C2・標的選定・初期認証情報は人間が用意。

なお個々のマイナー種別やペイロードのハッシュ等、記事に記載のない技術指標は**報道時点では未確認**です。

## 影響と被害範囲

- **対象は「インターネット公開された Langflow を運用する全組織」**。約7,000の公開インスタンス（主に北米）が潜在的な標的です。
- **リソース被害**：Monero マイナーによる計算資源の不正消費。
- **データ被害**：MySQL・Nacos の暗号化・削除。JadePuffer では**復号鍵が保存されず、データ復旧が不可能**という最悪の結果。
- **二次被害の連鎖**：窃取された OpenAI・Anthropic・AWS 等の API キーが悪用されれば、Langflow の外側にまで被害が波及します。AI 基盤が「認証情報のハブ」になっている点が被害を増幅させます。
- **脅威アクターの幅**：金銭目的の犯罪者（マイナー・ランサムウェア）から、昨年武器化した国家支援グループ MuddyWater まで。

## 防御・推奨アクション

実務者が今日から着手できる順に整理します。

1. **露出の遮断を最優先**：Langflow がパブリックにアクセス可能なら、直ちにインターネットから切り離すか、認証を強制する。管理パネル・API エンドポイントはネットワークレベルでアクセス制御（VPN／許可 IP／リバースプロキシ認証）を適用。これが最も確実な即効策です。
2. **バージョン更新**：報じられた修正状況に基づき、**少なくとも 1.9.1 以降**へ更新する。CVE-2025-3248 対応には 1.3.0 以降、CVE-2026-55255 対応には 1.9.1 が必要とされているため、最新の 1.9.1 系に上げるのが安全です。連邦機関でなくとも BOD 26-04 の期限（7月10日）を実質的な目安に。
3. **デフォルト設定の見直し**：Langflow の**未認証自動ログインを無効化**。MinIO 等のストレージで**デフォルト認証情報を使わない**。
4. **API キーの防御**：Langflow 内に保存された OpenAI・Anthropic・AWS 等のキーを**ローテーション**し、使用状況を監視、最小権限を徹底する。侵害が疑われる場合は即時失効。
5. **侵害の痕跡調査**：`/api/v2/files` への不審な POST、想定外のファイル書き込み、マイナー由来の CPU 高負荷、MySQL/Nacos への異常アクセスや削除、永続化タスクの有無を確認。
6. **公開 AI サービスの棚卸し**：Langflow に限らず、外部公開している AI 関連サービスのアクセス制御を監査する。

## 今後の見通し

Langflow の事例が示すのは、二つの構造的トレンドです。

第一に、**AI 開発基盤そのものが「価値の高い標的クラス」になった**こと。API キー・モデル・データが一点に集まる AI パイプラインは、攻撃者にとって費用対効果の高いターゲットです。脆弱性が連続して見つかり悪用されている現状を踏まえると、Langflow に対する攻撃圧力は当面続くと見るのが妥当です。

第二に、**攻撃側の自動化**です。JadePuffer は「初の自律型 LLM 駆動ランサムウェア」と評され、攻撃スキルの敷居を下げる可能性を示しました。ただし現時点では初期セットアップに人間が不可欠であり、「完全自律」ではありません。過度な脅威煽りは禁物です。とはいえ、31秒での戦術自己修正という適応力は、**将来的により人手の少ない攻撃**へ向かう方向性を示唆しています。

実務的な結論はシンプルです。**AI 基盤を「特別なアプリ」と見なさず、通常の外部公開システムと同じ規律——露出最小化・迅速なパッチ・認証情報管理・監視——で守ること**。Langflow の1か月は、その規律を怠った先に何が待つかを圧縮して見せた事例でした。
